PROJECT OVERVIEW
----------------
The Hotel Reservation System is a comprehensive desktop application designed
to streamline hotel operations and enhance guest experiences. Built using
JavaFX for the user interface and Oracle Database for data persistence, this
system provides a complete solution for managing hotel reservations, guest
information, room inventory, and financial transactions.

This project was developed as part of the CS313 course requirements and
demonstrates advanced object-oriented programming concepts, database
integration, and modern UI design principles.


PURPOSE AND OBJECTIVES
-----------------------
The primary objectives of this system are to:

1. Automate hotel reservation processes to reduce manual errors
2. Provide role-based access control for different user types
3. Enable efficient room inventory management
4. Facilitate guest self-service for booking and managing reservations
5. Generate comprehensive reports for business intelligence
6. Demonstrate practical application of core Java programming concepts


TARGET USERS
------------
The system is designed for three distinct user roles:

1. GUESTS
   - Search and book available rooms
   - View and manage personal reservations
   - Cancel bookings when needed
   - Access booking history

2. RECEPTIONISTS
   - Check guests in and out
   - Process payments
   - Manage room availability
   - Handle guest inquiries
   - View all system reservations

3. ADMINISTRATORS
   - Monitor system statistics and performance
   - Manage users (create, update, delete accounts)
   - Configure room types and services
   - Generate business reports (revenue, occupancy, guest analytics)
   - Oversee all system operations


KEY FEATURES
------------

RESERVATION MANAGEMENT
- Real-time room availability checking
- Date-based search with conflict prevention
- Automatic price calculation based on room type and duration
- Multiple reservation statuses: Pending, Confirmed, Checked-In, Checked-Out, Cancelled

ROOM MANAGEMENT
- Categorization by room types (Single, Double, Suite, etc.)
- Dynamic pricing based on room type and season
- Real-time status tracking (Available, Occupied, Maintenance, Reserved)
- Capacity and amenity information

USER MANAGEMENT
- Secure authentication and authorization
- Role-based access control
- User profile management
- Search and filter capabilities by role

FINANCIAL MANAGEMENT
- Integrated payment processing
- Multiple payment methods support
- Automatic receipt generation
- Revenue tracking and reporting

REPORTING & ANALYTICS
- Revenue reports with date range filtering
- Occupancy rate analysis
- Guest statistics and demographics
- Room status summaries
- Export functionality for business intelligence


TECHNICAL ARCHITECTURE
-----------------------

PROGRAMMING LANGUAGE
- Java SE 25 (compatible with Java 11+)
- Object-oriented design principles

USER INTERFACE
- JavaFX 21 for rich desktop UI
- FXML for declarative UI design
- Consistent color scheme: Dark Brown (#321F08) and Light Beige (#FAF8F3)
- Times New Roman font family for professional appearance

DATABASE
- Oracle Database 21c Express Edition
- JDBC for database connectivity
- Prepared statements for SQL injection prevention
- Connection pooling for performance

DESIGN PATTERNS
- Model-View-Controller (MVC) architecture
- Singleton pattern for database connections
- Factory pattern for user object creation
- Observer pattern for UI updates


JAVA CONCEPTS DEMONSTRATED
---------------------------

1. INHERITANCE
   - Abstract User class with concrete subclasses (Guest, Receptionist, Admin)
   - Code reusability and polymorphic behavior

2. POLYMORPHISM
   - Role-based method overriding
   - Dynamic method dispatch
   - Interface implementation (Comparable)

3. COMPOSITION
   - Room HAS-A RoomType relationship
   - Strong ownership and lifecycle management

4. COLLECTIONS FRAMEWORK
   - ArrayList for dynamic lists of objects
   - HashSet for ensuring username uniqueness
   - TreeSet for automatic sorting (room types by price)
   - LinkedList for queue operations
   - HashMap for key-value associations
   - Iterator for collection traversal

5. INTERFACES
   - Comparable interface implementation
   - Custom comparators for sorting

6. EXCEPTION HANDLING
   - Custom exception classes
   - Try-catch-finally blocks
   - Proper resource management
   - User-friendly error messages

7. JDBC (Database Connectivity)
   - Connection management
   - PreparedStatement for parameterized queries
   - ResultSet processing
   - Transaction handling
   - SQL aggregate functions (COUNT, SUM, AVG, GROUP BY)

8. FILE I/O
   - Reading configuration files
   - Exporting reports

9. JAVAFX UI COMPONENTS
   - TableView for data display
   - DatePicker for date selection
   - ComboBox for dropdown menus
   - Scene navigation and management


DATABASE SCHEMA
---------------

CORE TABLES:
- USERS: Authentication and role information
- GUESTS: Guest-specific details (name, ID, address)
- ROOMS: Room inventory with status and pricing
- ROOM_TYPES: Room categories and specifications
- RESERVATIONS: Booking records with dates and status
- PAYMENTS: Financial transaction records
- SERVICES: Additional hotel services (room service, spa, etc.)

RELATIONSHIPS:
- Users have a one-to-one relationship with role-specific tables
- Rooms belong to room types (many-to-one)
- Guests create reservations (one-to-many)
- Reservations reference rooms (many-to-one)
- Reservations have payments (one-to-one)
