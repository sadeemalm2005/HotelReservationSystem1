===============================================================================
                        KEY RELATIONSHIPS
===============================================================================

INHERITANCE (IS-A):
    User → Guest, Receptionist, Admin

COMPOSITION (HAS-A):
    Room ◆─── RoomType (Room cannot exist without RoomType)

ASSOCIATION:
    Guest ────── Reservation (1 guest creates 0..* reservations)
    Room ────── Reservation (1 room has 0..* reservations)
    Reservation ────── Payment (1 reservation has 1 payment)

DEPENDENCY:
    All Controllers ──use──> DatabaseConnection
    All Controllers ──use──> Model Classes


===============================================================================
                    JAVA CONCEPTS DEMONSTRATED
===============================================================================

1. INHERITANCE & POLYMORPHISM
   - User is abstract base class
   - Guest, Receptionist, Admin extend User
   - Role-based polymorphic behavior

2. COMPOSITION
   - Room HAS-A RoomType (strong ownership)
   - Room cannot exist without a valid RoomType

3. INTERFACES
   - RoomType implements Comparable<RoomType>
   - Enables sorting by price using TreeSet

4. COLLECTIONS (java.util)
   - ArrayList: Storing lists of users, reservations, rooms
   - HashSet: Ensuring unique usernames
   - TreeSet: Auto-sorting room types by price
   - LinkedList: Queue operations (if implemented)
   - Queue: Processing check-in/check-out requests

5. EXCEPTION HANDLING
   - Custom exceptions for specific scenarios
   - Try-catch blocks in all database operations
   - Proper resource cleanup in finally blocks

6. JDBC (java.sql)
   - Connection: Database connectivity
   - PreparedStatement: SQL queries with parameters
   - ResultSet: Processing query results
   - Proper resource management

7. JAVAFX (UI)
   - FXML for UI design
   - Controllers linked to FXML views
   - Event handling (@FXML annotations)
   - TableView for data display
   - Scene navigation


===============================================================================
                        DATABASE SCHEMA
===============================================================================

USERS (user_id, username, password, email, phone, role, created_date)
    ↓ 1:1
GUESTS (guest_id, user_id, first_name, last_name, id_number, address)

ROOM_TYPES (type_id, type_name, description, base_price, capacity)
    ↓ 1:N
ROOMS (room_id, room_number, type_id, status, price)
    ↓ 1:N
RESERVATIONS (reservation_id, guest_id, room_id, check_in_date,
              check_out_date, total_price, status, booking_date)
    ↓ 1:1
PAYMENTS (payment_id, reservation_id, amount, payment_method,
          payment_date, status)

SERVICES (service_id, service_name, description, price)


===============================================================================
                    DESIGN PATTERNS USED
===============================================================================

1. MVC (Model-View-Controller)
   - Models: User, Room, Reservation, Payment, etc.
   - Views: FXML files (Login.fxml, GuestDashboard.fxml, etc.)
   - Controllers: LoginController, GuestDashboardController, etc.

2. Singleton Pattern
   - DatabaseConnection manages single connection instance

3. Factory Pattern (Implicit)
   - User creation based on role (Guest/Receptionist/Admin)
